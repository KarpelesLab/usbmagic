LINKTYPE_USB_TYPE_C_PD

This is a proposal (the next currently-unallocated link-layer header type number when written was 304), following the guidance in libpcap issue #1036. It is implemented by the usbmagic USB forensics tool, which captures USB Power Delivery on the Configuration Channel (CC) of a USB Type-C connection.

Packet structure

+---------------------------+
|          version          |   = 1
|         (1 Octet)         |
+---------------------------+
|         sop_type          |   SOP* ordered set
|         (1 Octet)         |
+---------------------------+
|         direction         |   0 = from DUT, 1 = to DUT
|         (1 Octet)         |
+---------------------------+
|           flags           |
|         (1 Octet)         |
+---------------------------+
|           crc32           |   little-endian
|         (4 Octets)        |
+---------------------------+
|          Payload          |   Power Delivery message (§6.2)
.                           .
.                           .

Description

The pseudo-header is 8 octets. Multi-octet fields are in little-endian byte order.

The version field is the pseudo-header version; it is 1 for this revision.

The sop_type field identifies the Start-of-Packet ordered set (or ordered-set-like framing) the message was carried in: 0 = SOP, 1 = SOP', 2 = SOP'', 3 = SOP'_Debug, 4 = SOP''_Debug, 5 = Hard Reset, 6 = Cable Reset, 0xFF = unknown/not recorded.

The direction field is 0 if the message was sent by the Device Under Test toward the capturing instrument (received), and 1 if it was sent by the instrument toward the Device Under Test (transmitted).

The flags field is a bit field: bit 0 (CRC_PRESENT) is set if the crc32 field holds the actual on-wire CRC-32; bit 1 (CRC_VALID) is set, only when CRC_PRESENT is set, if that CRC was verified as correct; bit 2 (CC_POLARITY) is clear for CC1 and set for CC2; all other bits are reserved and set to zero. Controllers that compute and check the CRC in hardware (for example the FUSB302B, which performs "AUTO_CRC") do not expose the received CRC octets, so captures from such controllers clear CRC_PRESENT and set crc32 to 0.

The crc32 field is the on-wire CRC-32, in little-endian byte order, when CRC_PRESENT is set; otherwise it is 0.

The payload is a Power Delivery packet as specified by section 6.2 "Messages" of the Universal Serial Bus Power Delivery Specification, revision 3.1, version 1.1: the 16-bit Message Header, followed by any Extended Message Header and Data Objects. The 4b5b line coding, preamble, SOP ordered set, and trailing CRC/EOP of the physical layer are not included in the payload (the SOP and CRC are represented in the pseudo-header instead).
